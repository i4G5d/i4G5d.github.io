SQLI

做为作为基础，也最为重要的漏洞之一，这里总结SQL注入

SQLI原理：

  1. 形成原因：

    ​ 当web应用向后台数据库传递SQL语句进行数据库操作时，如果对用户输入的参数没有经过严格的过滤处理，那么攻击者就可以构造特殊的SQL语句，直接输入数据库引擎执行，获取或修改数据库中的数据。

  2. 本质：

    ​ 用户能控制输入的内容； web应用把用户输入的内容带入到数据库当代码执行；

SQL注入思路：

1. 判断可靠输入点

  1. GET/POST注入

    要判断我们传入的参数如何和数据库交互，判断如何闭合

      2. Cookie

    有些网站通过查询cookie判断用户是否登录，需要与数据库进行交互，我们可以修改cookie的值，查找我们所需要的东西。或者通过报错注入是网页返回报错信息。

      3. Referer注入

    有些网站会记录ip和访问路径，例如百度就是通过Referer来统计网站流量，我们将访问路径进行SQL注入，同样也可以得到想要的信息。

      4. XFF注入

    在用户登录注册模块在 HTTP 头信息添加 X-Forwarded-for: 9.9.9.9’ ，用户在注册的时候，如果存在安全隐患，会出现错误页面或者报错。从而导致注册或者登录用户失败。

      5. UA注入

2. 判断库类型

​ 常见数据库就是MYSQL(3306)，SQLserver(1443)，Oracle(1521)，SQLite(无),PostgreSQL(5432)。

​ 判断方法主要就是

  • 数据库特有的函数

    数据库 特有函数 示例 payload 判断逻辑
    MySQL version()、database()、sleep() ' UNION SELECT version()--+ 如果返回类似 8.0.33 → 确认是 MySQL
    PostgreSQL current_database()、pg_sleep()、VERSION() ' UNION SELECT pg_sleep(5)--+ 页面延迟 5 秒 → PostgreSQL
    SQL Server @@version、db_name()、waitfor delay ' UNION SELECT @@version--+ 返回 “Microsoft SQL Server” 或延迟 5 秒
    Oracle banner from v$version、dbms_pipe.receive_message ' UNION SELECT banner FROM v$version--+ 报错/回显包含 “Oracle Database”
    SQLite sqlite_version() ' UNION SELECT sqlite_version()--+ 返回 3.x.x

  • 数据库专属符号来判断，如注释符号、多语句查询符等等

    特征类型 MySQL SQL Server Oracle PostgreSQL SQLite
    单行注释 --+ 或 # --+ --+ --+ --+
    多语句执行 ; ; 不支持（默认） 支持 支持
    字符串连接符 concat('a','b') 'a'+'b' `’a’ ‘b’`

  • 报错信息

  • 其他数据库特性

​

3.判断参数类型

​ 通过+1，-1，and，or，注释符的组合进行刺探。（数值型，字符型，闭合类型）

  • id=1 and 1=1回显正常 id=1 and1=2回显错误（判断为整形）
  • id=1 and 1=1和id=1 and 1=2回显正常（判断为字符型接下来判断闭合方式）
  • id=1’ and ‘1’=’1回显正确 id=1’ and ‘1’=’2回显错误（判断为【’】闭合）
  • id=1” and “1”=”1 回显正常 id=1” and “1”=”2回显错误（判断为【”】闭合）
  • SQL语法支持一个或多个「括号」包裹参数，上述三种都可能带个括号。

4.判断过滤

已经确定的库类型，注入参数类型，就可以开始注入了，但过滤是肯定存在的。

  • 参数会有过滤
  • 没有显示位；就考虑报错注入或者时间注入
  • 没有报错信息；就不能报错注入

 	<summary>**关于将id值设置为0或者负数的解释**</summary>
 ​	由于我们的语句是插入到原有语句后面，这样就会出现两个SQL语句同时执行，由于SQL查询会默认返回一行数据，所以我们插入的第二行语句的结果就不会被返回，只会返回原有的SQL语句的查询内容。
 要让数据库查询我们插入的语句，需要让原有SQL语句产生查询错误，注意：查询错误不是语法错误，查询错误只会返回空，不会让语句报错。
 所以我们可以使id=0或id=-1，零或负数不会被用作id值，它插入进去一定导致原有SQL语句查询结果为空，我们插入的SQL语句的结果就会被返回。
 联合查询时union select无法执行时 用union+select代替 接下来的查询语句空格全部用【+】代替
