4.端口扫描

naabu暂未使用
端口扫描的核心逻辑是向目标 IP 的指定端口发送探测数据包，根据响应结果判断端口是否开放

  • 开放端口：会回复 “同意连接”（如 TCP SYN+ACK 包）；
  • 关闭端口：会回复 “拒绝连接”（如 TCP RST 包）；
  • 过滤端口（防火墙拦截）：无响应或回复 “不可达”。
    端口扫描要做的就是
  • 找存活（攻击面） 开放端口的 IP 地址 和 端口号。
  • 找信息（漏洞暴露） 端口上运行的 服务名称 和 版本号。
  • 服务器响应头中的 WAF、CDN 或 操作系统 指纹。

整理

  • 非 Web 服务： 将开放了非 80/443 端口的 IP 地址（如 1.1.1.1:3306）直接输入给 网络/系统级漏扫工具（如 Nessus）。

  • Web 服务： 将所有开放 80/443 端口的 IP 地址，以及通过 HTTPX 验活的 URL 列表，输入给 Web 应用漏扫工具（如 Awvs, Xray）。这些工具内部的漏洞库会根据 Nmap 提供的版本指纹，自动匹配是否存在可利用的 CVE。

MSF

在搜集到指纹信息或存在后门时尝试使用MSF渗透
Metasploit Framework（MSF）是一款开源渗透框架，该框架分为内核层，模块层，接口层

模块层是 MSF 最核心的部分，所有渗透测试操作都通过调用模块实现。

接口层用于调用这些模块，提供多种与框架交互的方式
msfconsole: 这是最常用、功能最全的命令行接口 (CLI)。
msfvenom: 一个独立的工具，用于生成和编码 Payload。 （附属工具）
Armitage: 可视化操作 MSF，拖放模块即可执行，适合新手入门
Cobalt Strike: 基于 MSF 内核的商业化扩展（非官方），支持团队协作、高级后渗透(企业级) （附属工具）

msfvenom

用于各种恶意软件和后门

  • -p 指定使用payload -p windows/meterpreter/reverse_tcp

    • {[OS]/[Architecture]/[Payload Type]/[Connection Method]}
    • os：windows/, cmd/windows/,linux/, cmd/unix/,android/,java/, php/
    • 目标架构 (Architecture): 默认32位(x86),脚本或命令payloads也是不用声明
    • Payload Type：meterpreter，shell，vncinject，add_user
    • Connection Method:reverse_tcp / reverse_http/reverse_php,bind_tcp

  • -f 指定输出格式 例如 exe、c、elf、php、js -f exe

  • -e 指定encoder -e x86/shikata_ga_nai

  • -a 指定架构 例如 x86(32位) x64(64位)-a x86

  • –platform 指定操作系统 -platform Linux

  • -s 指定最大长度

  • -b 指定规避字符

  • -i 指定编码次数

  • -x 绑定正常程序 `-x /root/notepad.exe

  • –shellest 最小化payload

  • lhost=192.168.1.20

  • lport=9999

  • -o

但exe容易被查杀，所以更多情况下生成shellcode代码(二进制字节流)，并通过三方应用或系统漏洞进行注入。
shellcode通过漏洞注入到目标系统的内存中，再劫持执行流让 CPU 执行它。

  • 缓冲区溢出 [ 垃圾数据（填满缓冲区） ] + [ 伪造的返回地址 ] + [ Shellcode ]
    意思就是本来只有100个字节的缓冲区 正常输入2字节 100字节后就是地址。但通过这个漏洞 攻击者注入200字节 导致cpu处理的时候把恶意注入的100到104或108误以为是返回地址，然后返回地址又指向最后的shellcode
  • 代码注入漏洞
  • 恶意文件载体。exe将code解压到内存
  • 进程注入 meterpreter提权后能进程注入
