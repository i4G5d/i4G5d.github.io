SRC 挖掘流程(通用 + 公益 SRC + 教育 SRC)

阶段 0：确认合法范围（非常重要）

  • 公益 SRC（如公益组织、社区项目）、教育 SRC（高校、科研单位）通常设有公开 SRC / VDP 范围。
  • 只有在允许范围内的域名 / 系统才可以进行主动测试（端口扫描、指纹探测、漏洞验证）。
  • 没写范围的，默认【只能被动信息收集】。

1：厂商信息与一级资产收集（OSINT）

1.1 通过工商信息类平台获取厂商基本资产

OSINT 的第一步：

  • 天眼查 / 企查查 / 爱企查 / 小蓝本
    用途：
    → 获取公司主体名称
    → 获取公司注册的主要官网域名
    → 获取实际控制人、关联企业（用于判断是否需要把子公司域名纳入资产）

1.2 从域名反查相关资产

  • whois / 站长工具（查看主域名、DNS、注册邮箱等）
  • 根据 注册邮箱、注册者名称 反查其他域名（一些厂商会使用同一个邮箱注册多个域名）

得到结果：初步的「一级域名列表」

2：二级 & 三级域名收集

2.1 被动子域名收集（首选，安全，不引发告警）

  • OneForAll：全面（国内资源丰富）
  • subfinder✨快
  • amass passive✨补充国外源
  • 连证书透明度（crt.sh）也能捞到大量子域
  • FOFA / Shodan / ZoomEye（后面解释）

这一步是「不主动攻击、不发请求，只是查公开信息」。

2.2 主动子域名爆破（视 SRC 范围而定）

  • 子域名收割机
  • Layer 子域名爆破
  • Goby 内置爆破
  • Amass 的 active mode

主动爆破属于 主动扫描，如果 SRC 范围没有允许扫描，会存在违规风险。
（公益 SRC / 高校 SRC 一般允许轻扫描，但你一定要看政策）

3. 资产测绘与 IP 收集

你会看到很多工具：“goby、fscan、xray、masscan、nmap”等

但你必须搞清楚它们的功能位置和顺序。

3.1 子域名转 IP

  • DNS 解析
  • 工具自带（OneForAll 自动解析）

得到：域名 -> IP 列表

3.2 发现网段

你说的站长工具、whois 查网段没问题，但要注意：

  • 不是所有同网段的 IP 都属于目标厂商
  • 不能因为在同网段就主动扫描整个 C 段
    （这个是很多初学者会违规的地方）

3.3 Shodan / FOFA / 微步在线

它记录了全球互联网上所有开放端口的服务信息、指纹、证书、域名、内容特征，并让你能按任何特征搜索。

这些网站是已经帮忙扫描过全球IP，我们是从他们的数据库中搜索已经暴露在网上的真实存货资产

它们提供的信息包括：

  • 暴露的端口

  • 服务/Web 指纹

  • Banner 信息

  • 证书信息

  • 厕所后台

  • API

  • 暴露的后台登录界面

  • 开发系统

  • 物联网设备

  • 老旧业务系统

    举例：你查某高校主域名，然后 FOFA 搜公司名/证书/子域名，即可自动发现：

    • 老旧 OA 系统
    • 宿舍管理系统
    • 图书馆系统
    • 内部开发系统曾经误暴露的测试域名
    • 历史暴露端口
    • 旧 IP 上已不再使用的后台界面
    • 误暴露的文件服务器、KMS、VPN、Zabbix、Jenkins、Nacos 等

    这些都是 SRC 挖掘时最容易出洞的资产来源。

⚠ 这比你自己扫安全得多、全面得多，因为它们属于“被动查询”，不是你自己在发扫描包。

到这里总结一下要搜集的东西 1.众多子域名（挖掘web逻辑漏洞）2. 域名转IP（服务端，框架漏洞，未授权访问）3.ip再反查或查绑定域名，不仅能做补充，还有捡漏的可能

域名告诉你哪里可能有 Web，IP 告诉你哪里还藏着没人注意的服务。
而“IP 反查域名”告诉你：厂商真正的攻击面，比你想象的大得多。

4. 端口扫描 & 指纹识别（属于主动探测）

大致的工具顺序应该这样用：

4.1 masscan（快速扫描大范围）

  • 快速找开放端口
  • 高速但危险，不建议直接扫公网上大量 IP，容易打爆厂商服务

4.2 nmap（细致扫描）

  • 对 masscan 发现的端口做指纹识别
  • 服务版本、操作系统识别等

4.3 goby/fscan/xray/pocsuite（综合指纹+弱密码+部分漏洞探测）

这些是高阶资产测绘工具，功能包括：

  • 自动识别 Web 服务
  • 自动识别后台路径
  • 自动识别框架（Spring、ThinkPHP、Struts 等）
  • 自动识别版本/组件
  • 判断是否存在历史已知漏洞

这一步是从资产转向漏洞的桥梁。

5：漏洞验证（手工为主）

有了目标资产后：

  1. 选看起来容易的系统（后台、老框架、测试环境）
  2. 尝试弱密码、逻辑漏洞、越权、任意文件读取、SQL 注入、XSS etc
  3. 使用 xray、Burp 等工具辅助分析
  4. 结合你学过的常见漏洞开始挖洞
