web_shell

这里记录一下几个常用webshell工具的流量特征

我们可以根据这些特征来判定webshell的存在，从而进行应急响应

菜刀/蚁剑/冰蝎/哥斯拉这四个都是web’shell管理工具，首先讲讲如何处理webshell（带一下其他shell）

  1. 扫描和确认
    看web服务器的日志，或者看流量特征，使用安全扫描工具确认webshell的存在
  2. 取证和删除
    保留证据后，进行删除，相关文件 日志 缓存什么都都删一下
  3. 加固
    针对攻击类型进行修复 打补丁，修复服务器代码和逻辑后，有权限和能力就加强监控和审计软件
  4. 善后
    修改一下服务器管理员的账号密码 防止已被攻破
工具 核心特点 流量加密方式 关键流量特征 WAF 绕过能力
中国菜刀 (Chopper) 最古老、最简单 明文 / 简单 Base64 eval、base64_decode、参数 z0、$_POST 极差 (几乎秒杀)
蚁剑 (AntSword) 菜刀的升级版 编码器混淆 (Base64/Hex) @ini_set、display_errors、User-Agent 明显、解码后可见代码 较差 (除非自定义编码器)
冰蝎 (Behinder) 加密隧道时代 AES / XOR 加密 密钥协商 (早版本)、强加密乱码、特殊的 Accept/UA 头 强 (流量无法解密)
哥斯拉 (Godzilla) 免杀之王 AES 加密 (强依赖 Cookie) Cookie 中含关键参数、响应包可能含空字节、Pass/Key 极强 (专为绕过 WAF 设计)

1. 中国菜刀 (China Chopper)

  • 地位： 入门级工具，现实中很少用，因为一用就被封。
  • 特征：
    • Payload 极短： 通常是一句话木马。采用url编码+base64编码，payload部分是明文传输。
    • 明文特征明显： 请求体中包含明显的 PHP/ASP 执行函数。最经典的是 eval(base64_decode($_POST['z0']))。菜刀只支持eval，post，base64，参数名也不支持更换
    • 参数名： 默认参数通常是 z0、z1、z2。
  • 简单总结： “菜刀的流量特征最明显，主要是基于 HTTP POST，Payload 中经常能看到 eval、base64 ，Z0等关键字，容易被 WAF 识别。”（默认UA是百度，火狐）

2. 蚁剑 (AntSword)

  • 地位： 菜刀的开源、模块化升级版，支持自定义编码器，ctf最常用。

  • 特征：

    • User-Agent： 默认情况下，User-Agent 可能含有 antSword/v2.0 等字样（虽然很容易改）。
    • Shell 代码特征： 每个请求体（Body）前面通常会带有一段用于设置环境的代码， @ini_set("display_errors", "0"); @set_time_limit(0);。环境配置代码，关闭错误显示和取消执行时间限制
    • 编码： 虽然使用了 Base64，但解码后能看到清晰的 PHP 代码结构。(默认编码)
      这是没编码的。响应体的返回结果是base64编码发混淆字符，格式为：随机数 结果 随机数。 蚁剑会在编码前加两位随机生成的字符，所以需要调整开始位置

  • 简单总结： “蚁剑比菜刀多了一些混淆，但默认情况下 User-Agent 有特征，且请求体中常包含 ini_set、set_time_limit 等配置 PHP 环境的函数代码。默认base64编码

3. 冰蝎 (Behinder)

  • 地位： 划时代的工具，引入了“动态二进制加密”，让流量看起来像乱码。
  • 流量体： 请求和响应全是乱码（AES 加密），无法通过关键字匹配。
  • 进行请求时内置十几个UA

3.1 冰蝎2.0

  • 特征：
    • 连接时有一个明显的密钥协商过程（Key Exchange）。攻击者先发随机字符串请求获取密钥，服务器返回密钥（互相生成动态密钥），后续流量才对称加密。这个“协商过程”和是探测点。
    • 进行请求时内置十几个UA，一个ip一直换UA。

3.2 冰蝎3.0

  • 特征：

    • 采用了预共享密钥（连接密码的MD5前16），没有协商过程，直接发加密流量。

      默认连接密码是”rebeyond”(即密钥是md5(‘rebeyond’)[0:16]=e45e329feb5d925b)

    • Content-Type：application/octet-stream （3.0，jspwebshell）

    • 进行请求时内置十几个UA，一个ip一直换UA。

3.3 冰蝎4

  • 特征：

    • 4.0+： 采用了预共享密钥（密码的MD5前16），没有协商过程，直接发加密流量。

      默认连接密码是”rebeyond”(即密钥是md5(‘rebeyond’)[0:16]=e45e329feb5d925b)

    • Content-Type： Application/x-www-form-urlencoded（4.）

    • Accept: application/json, text/javascript, /; q=0.01 意思是浏览器可接受任何文件，但最倾向application/json 和 text/javascript。

    • UA：内置十几个UA，一个ip一直换UA

    • connetion长链接 请求和响应会有Connection：Keep-Alive

    • 固定代码 ：有固定的请求头和响应头，请求字节头：dFAXQV1LORcHRQtLRlwMAhwFTAg/M ，响应字节头：TxcWR1NNExZAD0ZaAWMIPAZjH1BFBFtHThcJSlUXWEd

  • 简单总结： “冰蝎的主要特征是全加密流量。早期版本有密钥协商过程，现在主要是通过分析流量的熵值（随机性）和非正常的 HTTP Header 组合来检测。

4. 哥斯拉 (Godzilla)

  • 地位： 目前最难对付的工具之一，专为绕过 WAF 和流量监测设计。

  • 特征：

    • Cookie 依赖： 哥斯拉非常依赖 Cookie 来传递 Session 或关键参数。最后会有个分号
    • 响应包特征： 响应体的数据有一定特征，把一个32位的md5字符串按照一半拆分，分别放在base64编码的数据的前后两部分。整个响应包的结构体征为：md5前十六位+base64+md5后十六位。 早期版本的响应包如果不含数据，可能会返回一个空的 HTTP 200，或者特定的头部结构。
    • Accept默认是Accept:text/html, image/gif, image/jpeg, *; q=.2, /; q=.2。同上，这个也可修改，只能作为辅助检测的特征。

  • 简单总结： “哥斯拉也是全加密，但它倾向于在 Cookie 中存放加密数据。它的 JSP Payload 会自带一些特定的 Java 类加载逻辑，且响应包有时候会有特定的长度特征。”

5.msfconsole&msfvenom

用于各种恶意软件和后门，攻击手段首选meterpreter，后shell，shellcode比shell.exe更能隐藏和防查杀。尽量与其他合法exe结合或者写入内存。但这些都不是webshell。

  • -p 指定使用payload -p windows/meterpreter/reverse_tcp
    • {[OS]/[Architecture]/[Payload Type]/[Connection Method]}
    • os：windows/, cmd/windows/,linux/, cmd/unix/,android/,java/, php/
    • 目标架构 (Architecture): 默认32位(x86),脚本或命令payloads也是不用声明
    • Payload Type：meterpreter，shell，vncinject，add_user
    • Connection Method:reverse_tcp / reverse_http/reverse_php,bind_tcp
  • -f 指定输出格式 例如 exe、c、elf、php、js -f exe
  • -e 指定encoder -e x86/shikata_ga_nai
  • -a 指定架构 例如 x86(32位) x64(64位)-a x86
  • –platform 指定操作系统 -platform Linux
  • -s 指定最大长度
  • -b 指定规避字符
  • -i 指定编码次数
  • -x 绑定正常程序 `-x /root/notepad.exe
  • –shellest 最小化payload
  • lhost=192.168.1.20
  • lport=9999
  • -o

msfconsole连接后可用session管理会话
