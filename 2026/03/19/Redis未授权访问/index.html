Redis未授权访问

前置

Redis是一种内存化键值对（Key-Value）数据库，默认端口 6379。常用于缓存，会话存储，特点是速度快，支持多种数据结构（曾经做课程项目也试过用redis存储邮箱验证码）

  1. 默认无认证机制
    任何人只要能网络访问到 6379 端口，即可直接连接并执行所有命令（如读写数据、修改配置、甚至执行系统命令）。默认配置：requirepas注释状态

  2. 数据持久化机制
    RDB：默认开启，在指定时间间隔内将内存中的数据快照写入磁盘（默认文件为 dump.rdb）
    AOF：记录所有写命令到日志文件（默认文件为 appendonly.aof），重启时重新执行命令恢复数据，将磁盘的数据写回内存。

  3. 系统命令执行
    Redis 可通过 module load 加载自定义模块（.so 文件），部分模块（如 redis-rogue-server）提供 system.exec 等命令，允许直接执行系统命令。（4.x 及以上支持模块加载）

  4. 配置动态修改 （

    命令）

    Redis 允许通过命令动态修改配置（无需重启服务），例如：

    • config set dir /root/.ssh/：修改持久化文件的存储目录；
    • config set dbfilename authorized_keys：修改 RDB 文件名；
    • config set appendfilename webshell.jsp：修改 AOF 文件名。
    • set 参数名 内容
      攻击者通过修改配置，将 Redis 持久化文件写入到服务器的敏感路径（如 SSH 公钥目录、Web 目录），实现权限提升或文件写入。

  5. 绑定 IP 配置（bind 选项）
    Redis 配置文件中的 bind 选项用于限制可访问的 IP 地址（默认 bind 127.0.0.1，即仅本地可访问）。若管理员将其修改为 bind 0.0.0.0（允许所有 IP 访问），则 Redis 服务会暴露在公网或局域网中。

Redis 默认生成的持久化文件（RDB/AOF）是 644 权限（所有者可读可写、同组用户可读、其他用户可读）
Ubuntu 对 “定时任务文件（crontab）” 有严格的权限要求 —— 必须是 600 权限（仅所有者可读可写，其他人无任何权限），否则 cron 服务会拒绝执行该定时任务，并报 “不安全模式” 错误。

4. 主从复制

应急与防护

  • 特征： 端口扫描 6379，流量中看到 Redis 的命令（config set, save, flushall）。
  • 绑定本地登录地址：在配置文件中设置 bind 127.0.0.1，限制仅本地访问
  • 设置复杂密码：取消requirepass 的注释，设置密码，防止密钥爆破
  • 禁用远程连接：设置 protected-mode 和 rename-command 防止远程连接
  • 网络隔离：将 Redis 服务放在内网环境，不向公网开放端口
  • 账号权限控制：设置只允许特定账号从指定IP进行访问
  • 实时监控：监控并阻止异常访问和命令执行

总结

就是利用Redis的默认无密码或者弱密码，导致攻击者可以通过gopher等协议控制redis
通常做的就是 config set dir，confg set dbfile，修改RDB的写入目录和具体文件，最终写入shell or SSH公钥 or 计划任务。
