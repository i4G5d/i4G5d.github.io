WebLogic_xml_T3反序列化

WebLogic

介绍

WebLogic是个巨大的中间件漏洞，由Java反序列化引起
Weblogic是Oracle 的中间件（比tomcat更强大），企业内网中，它通常位于 Web 服务器（如 Nginx）和数据库（Oracle DB）之间。。端口通常是 7001。
内网大杀器： 很多时候，外网打点进去后，内网扫描发现一台 WebLogic，基本就意味着拿下了这台服务器的权限。

1. XMLDecoder 反序列化 (HTTP )

  • 原理： 利用服务器的XMLDecoder可以解析XML数据这一特性，可以构造XML语句，发送一个 XML 格式的 HTTP POST 请求，上传一句话木马或者反弹shell，WebLogic 解析这个 XML 时执行了恶意代码，getshell。
  • 利用难度： 低（直接发包即可）。
    访问/wls-wsat/CoordinatorPortType（只要是wls-wsat包中的Url皆收到影响）
    构造XML，其中Content-Type需要等于text/xml，否则可能导致XMLDecoder不解析。
  • 适用路径

修改Conten-Type头为text/xml,写入恶意XML代码，响应包的状态码是500就成功了

随后nc监听nc -lvnp 6666就拿到shell了

2. T3 反序列化:

  • T3协议是Oracle Weblogic独有的（RMI的特殊利用），用于远程方法调用的专有协议。运行在客户端的java程序直接调用服务器上的java对象的method。 此协议运行在TCP之上，复用Weblogic主端口（7001）

原理


weblogic.rjvm.InboundMsgAbbrev#readObject 反序列化时，依赖的 resolveClass 方法无任何类名校验—— 允许反序列化任意 Java 类（包括恶意构造的类）。

  • 客户端发起类似t3 12.2.1 AS:255 HL:19 MS:10000000的握手包，服务器支持的话会回复类似HELO:12.2.1.3.0 false，随后双方开始传输序列化后的java对象数据。这也是最显著的漏洞特征

利用过程

  • 原本是客户端把正常的Java对象序列化成二进制流，通过T3协议发给服务器，服务器反序列化处理业务
  • 攻击者利用 T3 协议，发送了一个恶意的序列化数据包，这个数据包里包裹的可能是CommonsCollections 利用链
  • 或者通过「JRMP 协议反向拉取恶意序列化代码」
  • Weblogic不检查装的是什么对象就进行反序列化，从而导致恶意利用链在内存里执行从而导致RCE
    利用手法一般就是ysoserial开启JRMP，并构造利用链

随后用exp向肉鸡后端发payload，让肉鸡去连接恶意JRMP

最后就可以看到监听端口有反弹shell了

3. 后台弱口令 / 未授权访问:

  • 后台地址： /console，/console/login/LoginForm.jsp
  • 常见弱口令： weblogic / Oracle@123, system / password。
  • 有时候可能无法直接登录访问http://Target:7001/console/console.portal，但通过二次编码的目录穿越（WebLogic 的权限检查器没认出穿越），http://Target:7001/console/css/%252E%252E%252Fconsole.portal,绕过了登录步骤

Weblogic反序列化（CVE-2023-21839）漏洞 https://www.cnblogs.com/BlogVice-2203/p/17454727.html
由于Weblogic IIOP/T3协议存在缺陷，当IIOP/T3协议开启时，允许未经身份验证的攻击者通过IIOP/T3协议网络访问攻击存在安全风险的WebLogic Server，漏洞利用成功WebLogic Server可能被攻击者接管执行任意命令导致服务器沦陷或者造成严重的敏感数据泄露。
WebLogic 在早期修复 T3 和 IIOP 的反序列化都是采用黑名单的方式修复 https://www.freebuf.com/vuls/345823.html

总结

  • Weblogic是一个7001中间件
  • 但XMLDecode可以解析XML，所以在反序列化XML中写入反弹shell等恶意代码
  • 此中间件用基于RMI协议的T3协议，让客户端的java程序直接调用服务器上的java对象的method
    所以利用服务器上的有Serializable 接口的类，用ysoserial写一个恶意的序列化文件
    用特定的exp文件向weblogic服务器发T3协议的文件
    服务器对文件进行反序列化就能导致恶意命令执行

ysoserial 的主要价值在于它内置了大量的 Java Gadget Chain（例如：CommonsCollections1、JRMPClient、Jdk7u21 等）。
